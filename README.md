# Obsidian Custom Sort（自定义排序插件）

> Obsidian 文件管理器自定义排序插件中文文档

---

## 功能特性

### 核心功能

1. **配置驱动排序** - 通过 YAML 配置文件精确控制排序顺序
2. **拖拽排序** - 在自定义排序视图中直接拖拽调整顺序
3. **多级排序规则** - 支持按文件夹、文件、修改时间、创建时间等排序
4. **通配符支持** - 使用通配符匹配文件名模式
5. **分组排序** - 按前缀、后缀或元数据对文件进行分组排序

### 拖拽排序视图（新增）

插件提供了一个独立的"自定义排序"视图，支持：

- **可视化拖拽排序** - 直接在树形视图中拖拽文件/文件夹
- **实时排序预览** - 拖拽时实时显示排序效果
- **自动保存** - 拖拽后自动更新 `sortspec.md` 文件
- **双向同步** - 同时支持 Obsidian 内置文件列表和自定义视图

## 快速开始

### 安装插件

1. 在 Obsidian 中打开 **设置** → **社区插件**
2. 搜索 "Custom File Explorer Sorting"
3. 安装并启用插件

### 创建排序配置文件

在任意文件夹中创建 `sortspec.md` 文件：

```yaml
---
sorting-spec: |
    target-folder: .
    < a-z
---
```

> **注意**：
> - `target-folder: .` 表示对当前文件夹应用排序规则
> - `< a-z` 表示按字母升序排序
> - `> a-z` 表示按字母降序排序

### 启用排序

点击 Ribbon 区域（左侧边栏底部）的插件图标：
- 图标变为绿色表示排序已启用
- 点击可切换启用/禁用状态

## 拖拽排序视图使用方法

### 打开视图

1. 点击 Ribbon 中的插件图标，或使用命令面板
2. 选择 "打开自定义排序视图"

### 视图功能

| 操作 | 说明 |
|------|------|
| **拖拽排序** | 拖动项目到两个项目之间，调整显示顺序 |
| **拖入文件夹** | 拖动项目到文件夹下方区域，移动到该文件夹 |
| **展开/折叠** | 点击文件夹左侧箭头展开/折叠子项目 |
| **应用排序** | 将当前视图中的排序应用到 Obsidian 文件列表 |
| **刷新目录** | 从磁盘重新读取排序配置并刷新视图 |
| **恢复默认** | 删除排序配置文件，恢复默认排序 |

### 排序规则

1. **同文件夹内拖拽** = 排序操作，调整顺序
2. **拖到展开的文件夹上方/下方** = 移动操作，移入该文件夹
3. **跨文件夹拖拽** = 移动操作

## 排序配置语法

### 基本语法

```yaml
---
sorting-spec: |
    target-folder: .   # 目标文件夹（. 表示当前文件夹）
    项目1               # 按名称显式排序
    项目2
    < a-z              # 剩余项目按字母升序排列
---
```

### 排序选项

| 语法 | 说明 |
|------|------|
| `< a-z` | 按字母升序（自然排序） |
| `> a-z` | 按字母降序 |
| `< true a-z` | 按字母升序（严格字母排序） |
| `> true a-z` | 按字母降序（严格字母排序） |
| `< modified` | 按修改时间升序（旧到新） |
| `> modified` | 按修改时间降序（新到旧） |
| `< created` | 按创建时间升序 |
| `> created` | 按创建时间降序 |
| `< advanced modified` | 按修改时间升序（包含文件夹内容） |
| `> advanced modified` | 按修改时间降序（包含文件夹内容） |

### 高级匹配

```yaml
---
sorting-spec: |
    target-folder: /*   # /* 表示应用到所有文件夹
    > advanced modified
---
```

```yaml
---
sorting-spec: |
    target-folder: .
    文件名...           # 匹配以"文件名"开头的项目
    项目A
    项目B
---
```

## 文件结构

```
vault/
├── sortspec.md                    # 根目录排序配置
├── folder1/
│   ├── sortspec.md                # folder1 排序配置
│   └── file1.md
└── folder2/
    └── sortspec.md                # folder2 排序配置
```

### 配置查找规则

1. 每个文件夹的排序配置存储在该文件夹的 `sortspec.md` 中
2. `sortspec.md` 只记录当前目录下的直接子项（文件和文件夹）
3. 不递归包含子文件夹的内容
4. 子文件夹的排序由各自的 `sortspec.md` 管理

## 常见问题

### Q: 拖拽后文件列表没有更新？

A: 请尝试以下步骤：
1. 点击"刷新目录"按钮
2. 如果仍未更新，尝试点击"应用排序"按钮
3. 重启 Obsidian

### Q: 子文件夹的排序没有生效？

A: 确保子文件夹中也有 `sortspec.md` 文件，且文件名与目录名称一致（不含 `.md` 后缀）。

### Q: 如何重置某个文件夹的排序？

A: 在该文件夹中删除 `sortspec.md` 文件，然后点击"恢复默认"按钮。

## 技术细节

### 排序配置存储

- **位置**: `{文件夹路径}/sortspec.md`
- **格式**: YAML frontmatter
- **内容**: 仅包含当前文件夹的直接子项

### 自动更新机制

1. 拖拽排序完成后，立即更新对应文件夹的 `sortspec.md`
2. 跨文件夹操作时，同时更新源文件夹和目标文件夹的配置
3. "刷新目录"会重新从磁盘读取所有展开文件夹的配置
